# **常见提示词误区与改进方法**

在学习编写提示词的初期，我们很容易陷入一些常见的误区。了解这些误区并掌握相应的改进方法，能帮助我们少走弯路，更快地提升与 LLM 的沟通效率。


| **常见误区**                                | **不良提示示例**                                                                                                                                                            | **改进后提示示例**                                                                                                                                                                                                                                                                        | **改进思路说明**                                                                                                                                                                                                                             |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. 过于模糊或宽泛**                   | “写点关于营销的东西。”                                                                                                                                                    | “为一家新开的社区有机面包店，撰写一篇 500 字左右的微信公众号推文，重点介绍其使用天然酵母、手工制作的特点，目标是吸引周边居民前来购买。请包含一个引人入胜的标题和至少两张面包的图片描述（用于配图）。”                                                                                   | **明确具体化**：清晰指出营销的具体载体（微信推文）、产品（有机面包）、卖点（天然酵母、手工）、目标受众（社区居民）、篇幅、以及对标题和图片描述的额外要求。这直接应用了“清晰具体”原则。                                                     |
| **2. 缺乏足够的上下文**                 | （在讨论了 A 方案后，又讨论了 B 方案，然后问：）“它有什么缺点？”                                                                                                          | “我们刚才讨论了营销方案 B（即通过社交媒体网红推广的方案）。关于这个方案 B，它在执行成本和效果可衡量性方面，可能存在哪些潜在的缺点或挑战？”                                                                                                                                              | **明确指代与补充背景**：清晰指明“它”具体是哪个方案，并限定了探讨缺点的维度（执行成本、效果可衡量性），为模型提供了必要的上下文信息，应用了“提供上下文”原则。                                                                             |
| **3. 提示过载，信息过多或指令冲突**     | “帮我写一个关于太空旅行的短篇故事，要非常科幻，但也要非常写实，适合儿童阅读，同时也要能启发成年人思考，字数在 300 字左右，但故事情节要曲折复杂，语言风格要既幽默又严肃。” | （第一步）“请写一个关于未来孩子们进行短途太空旅行的科幻小故事，字数 300 字左右，故事要充满想象力，语言生动有趣，适合 8-10 岁儿童阅读。”<br /> （第二步，若需）“基于上面的故事，能否扩展成一个约 800 字的版本，加入一些能引发成年人对科技与伦理关系思考的元素，同时保持故事的科幻基调？” | **分解任务，避免矛盾**：将一个包含过多甚至相互矛盾指令的复杂任务，分解成更小、更专注的子任务，分步提问。优先处理最重要的指令，避免让模型在冲突的目标间“左右为难”。这体现了结构化沟通和逐步求精的思路。                                     |
| **4. 期望一次完美，不进行迭代**         | （模型给出的第一个答案不理想，用户便认为模型不行，直接放弃。）                                                                                                              | （初次提示）“给我推荐几款适合健身的 App。”<br />*模型输出宽泛后，迭代提示*：“我是一个刚开始健身的初学者，主要想减脂和塑形，平时工作较忙，希望 App 能提供短时高效的家庭锻炼课程，并且有清晰的动作指导和激励机制。请基于这些需求，推荐 3 款具体的 App，并简述理由。”                      | **接受不完美，持续优化**：将模型的首次输出视为一个起点或初稿，分析其与期望的差距，然后通过修改、补充、细化提示词来进行迭代，逐步引导模型给出更满意的答案。这正是“迭代优化”原则的应用。                                                     |
| **5. 忽视模型限制，提出不切实际的要求**  | “请预测下周股票 X 的精确涨跌幅度。”<br /> “你真的有自我意识吗？你快乐吗？”                                                                                                | （针对知识截止）“根据你截止到[模型知识截止日期]的知识，过去五年中，人工智能在医疗诊断领域有哪些重要的应用突破？”<br /> （针对能力边界）“请帮我分析一下当前关于人工智能意识的几种主要哲学观点及其论据。”                                                                                 | **认知边界，合理提问**：了解并尊重 LLM 的基本工作原理和固有局限性（如无法预测未来、没有真实情感或意识、知识库有更新截止日期等）。将 LLM 视为强大的信息处理和模式生成工具，而非全知全能的先知或拥有人类情感的个体。提问时，应在其能力范围内。 |
| **6. 使用不礼貌或过于随意的语言**           | “赶紧的，随便给我编个笑话。”                                                                                                                                              | “请创作一个关于程序员和咖啡的简短笑话，风格轻松幽默。”                                                                                                                                                                                                                                  | **保持清晰和尊重（即使模型不在意）**：虽然有研究表明对 LLM 无需使用礼貌用语，但对于我们而言，清晰、结构化的指令通常比含糊不清、过于口语化的表达能带来更好的结果。明确的指令有助于模型准确理解任务，而过于随意的语言有时可能引入歧义。  |

通过对这些常见误区的分析，我们可以发现一个规律：**避免这些误区的核心，实际上就是更好地理解和应用前面章节中讨论的那些核心提示原则**。

* “过于模糊或宽泛”（误区 1）本质上是违背了“清晰具体”原则。
* “缺乏足够的上下文”（误区 2）则是没有遵循“提供上下文”原则。
* “提示过载或指令冲突”（误区 3）可以看作是在“清晰具体”上出了问题（指令不清晰导致冲突），或者是未能将复杂任务有效分解，这与结构化沟通的理念相悖。
* “期望一次完美，不进行迭代”（误区 4）显然是没有应用“迭代优化”原则。
* “忽视模型限制，提出不切实际的要求”（误区 5）意味着用户没有根据 LLM 的已知能力来调整其提示，这可以理解为未能提供关于 LLM 能力边界的现实“约束”或“上下文”。

因此，与其死记硬背各种“不要做”，不如将精力更多地投入到理解和实践那些基础的核心提示原则上。当遇到不理想的输出时，可以回过头来审视自己的提示词是否在清晰度、上下文、约束条件等方面存在不足，并据此进行调整。这种基于原则的自我纠错能力，是提升提示词工程水平的关键。
